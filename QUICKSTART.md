# BioLens Quick Start Guide

Get up and running with mechanistic interpretability for biomedical LMs in 5 minutes.

## Installation

```bash
git clone https://github.com/spoddar2008/biolens.git
cd biolens
pip install -r requirements.txt
```

**GPU (recommended):** Requires CUDA-capable GPU for efficient model loading.

**CPU-only:** Will work but be slow. Reduce batch_size in configs.

## First Run: 3 Steps

### Step 1: Explore the Model (2 min)
```bash
jupyter notebook notebooks/01_model_exploration.ipynb
```
This loads BioGPT and extracts activations from your first prompts.

**What you'll see:**
- Activation statistics across layers
- Layer-wise dimensionality analysis
- How activation magnitudes change with depth

### Step 2: Probe for Biological Concepts (10 min)
```bash
jupyter notebook notebooks/02_probing_classifiers.ipynb
```
This is THE key experiment. Linear probes reveal where biological knowledge emerges.

**Key insight:** Drug mechanism classification jumps from ~50% (random) to 90%+ at layer 18. This is where abstract biological understanding forms.

### Step 3: Find Safety Directions (5 min)
```bash
jupyter notebook notebooks/03_safety_analysis.ipynb
```
Finds directions in activation space that separate benign from sensitive content.

**What it means:** If internal activations can flag dangerous knowledge, BCIs might detect adversarial attempts harder than text-level filtering.

| Notebook | Time | Output | Purpose |

## What To Do Next

### 1. Expand the Prompt Dataset
Add more drug-target pairs and safety-relevant prompts to `data/prompts/`:
```json
{"text": "Your prompt here", "mechanism": "kinase_inhibitor", ...}
```

The more prompts, the more robust the probing results.

### 2. Analyze Attention Heads
Modify notebook 02 to extract which attention heads are most important:
```python
# In probes.py, use patching.py to trace which heads matter
patcher = ActivationPatcher(model)
result = patcher.run(
    clean="Imatinib inhibits BCR-ABL",
    corrupted="Imatinib inhibits RANDOM"
)
```

### 3. Test on Larger Models
Switch `model_config.yaml`:
```yaml
model:
  name: "PubMedBERT"  # or any HuggingFace biomedical model
```

Larger models may show different emergence patterns.

### 4. Publish
Write up your findings and submit to NeurIPS/ICLR workshop on:
- Mechanistic interpretability
- AI safety evaluation
- Biological applications of deep learning

## Troubleshooting

**Q: Out of memory during model loading?**
- Set `device: "cpu"` in `configs/model_config.yaml`
- Or use a smaller model: `microsoft/biogpt` → `allenai/smiles-bert-base-cased`

**Q: TransformerLens not installing?**
```bash
pip install transformer-lens --no-deps
pip install einops transformer-circuits
```

**Q: Activations are all zeros?**
- Check that your prompts are actually being tokenized
- Verify model is in eval mode (check `models.py`)

**Q: Probes give random accuracy (~50%)?**
- May need more training data
- Try increasing `C` parameter in BiologicalProbe
